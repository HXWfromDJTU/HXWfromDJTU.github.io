REST相关概念


  • REST ：

    • REST ( REpresentational State Transfer ) “表现层状态转化”。

    • 定义了REST的规范，主要说明了在web项目中，所有的东西都是资源，肯定都能使用URL来访问到。



  • RESTful：满足上述REST规范，所编出来的应用程序，都可以被称之为RESTful，是这一类程序的总称。

  • restlet 与 resteasy：是由实际的web服务器厂商生产出来的RESTful程序，又称之为RESTful web框架，实际就是一个servlet，通过在web.xml中配置url匹配，达到拦截url的效果。


REST作用



  • 对url进行规范。使url简洁。

  • 对http方法，进行规范。（官方声明，没什么卵用）

    • GET            获取资源

    • POST　　　  提交资源

    • PUT 　　　　更新资源

    • DELETE　　   删除资源



  • 对请求结果返回类型进行规范。（json格式的支持）
